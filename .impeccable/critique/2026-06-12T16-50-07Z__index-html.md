## Anti-Patterns Verdict

**LLM 평가**: "AI가 만들었다"고 단정할 표면은 아니다. 자갈돌 아바타와 일관된 일러스트 썸네일이 개성을 만들고, 금지 패턴(사이드 스트라이프, eyebrow, 히어로 메트릭, 동일 카드 반복) 없음. 다만 첫인상 골격은 여전히 Minimal Mistakes 실루엣(마스트헤드+좌 프로필+우 TOC)이고, 타이포 보이스가 없음 — 전체가 Nanum Gothic 단일 서체(2010년대 한국 블로그 기본값)로, "테마 위의 정체성" 원칙의 마지막 빈칸.

**결정적 스캔**: CLI(detect.mjs, `_layouts` `_includes` `_pages` `index.html`) — 발견 0건, 클린. 브라우저 디텍터(홈+포스트 주입):
- line-length ×26 (포스트 본문 ~100 latin-equiv chars/line, 권장 <80)
- low-contrast ×2: 흰 텍스트 on #11999e 3.5:1 (TOC 헤더 `h4.nav__title`, 페이지네이션 현재 페이지 `a.current`)
- tiny-text: 사이드바 bio 10.2px (`.author__bio p`)
- gray-on-color: 푸터 #40514e on #30e3ca (대비 5.2:1로 AA 통과, 색상 washout 지적 — 스타일 판단 사안)
- cramped-padding: 코드 블록 우측 인셋 없음
- skipped-heading ×2: h1→h4("On This Page"), h2→h4("공유하기")
- **거짓 양성**: gradient-text, theater-slop-phrase — 디텍터 자기 오버레이 배지 매칭으로 확인(페이지에 background-clip:text 요소 0개). monotonous-spacing은 오버레이 영향 가능성 있는 저신뢰 신호.

LLM 리뷰와 디텍터 합치점: 줄 길이/행간(LLM: 한국어 50자/줄·행간 1.5 ↔ 디텍터: line-length ×26), 단일 서체, 플랫 타입 스케일. 디텍터가 추가로 잡은 것: mint 배경 위 흰 텍스트 3.5:1, 10.2px bio, skipped headings.

## Overall Impression

정돈 작업이 잘 된, 신뢰감 있는 기술 블로그. 접근성 기본기와 패턴 일관성은 개인 블로그 평균을 한참 웃돈다. 가장 큰 기회는 사이트의 존재 이유인 **장문 읽기 경험 그 자체** — 본문 행간 1.5에 한 줄 50자(한국어 기준)는 "본문이 왕" 원칙과 어긋나는 마지막 미해결 영역이다.

## What's Working

1. **박스 없는 타이포 중심 카드 그리드** — 16:10 썸네일·라운드·호버 패턴이 홈/카테고리/관련글에 동일 적용. 차분하고 빠르다.
2. **접근성 기본기** — 스킵 링크, mint 포커스 링, 본문 대비 8.4:1, reduced-motion 대응, 보강된 터치 타깃.
3. **읽기 보조 장치** — 스티키 TOC 활성 추적, 카테고리 내 이전/다음, 검색 즉시 결과(한국어 placeholder).

## Priority Issues

1. **[P1] 한국어 장문 조판이 빽빽하고 넓다**
   - What: 본문 17px/행간 1.5, 본문 폭 846px ≈ 한 줄 50자(한국어). 디텍터도 동일 단락 26곳 지적. 코드 블록은 12.75px로 과소.
   - Why: 검색 유입 독자가 글을 끝까지 읽게 하는 것이 성공 기준인데, 핵심 표면의 읽기 피로가 가장 크다.
   - Fix: `line-height` 1.7~1.75, 본문 컬럼 최대폭 ~42em(약 700px), 코드 블록 14~14.5px + 우측 인셋.
   - Command: /impeccable typeset

2. **[P1] 404가 막다른 길**
   - What: 영어 기본 문구("Page not found :("), 검색·홈·추천 글 링크 0개.
   - Why: 검색/외부 링크 유입 블로그에서 깨진 링크 도착자는 그대로 이탈한다.
   - Fix: 한국어 안내 + 검색창 + 인기/최근 글 4개 + 홈 링크.
   - Command: /impeccable harden

3. **[P2] mint 배경 위 흰 텍스트 AA 미달 + 10.2px bio**
   - What: TOC 헤더·페이지네이션 현재 페이지 3.5:1(필요 4.5:1), 사이드바 bio 10.2px.
   - Why: WCAG AA 목표 선언(PRODUCT.md)과 직접 충돌.
   - Fix: mint를 어둡게(#0d7377급) 또는 텍스트를 진한 잉크로; bio ≥12.8px.
   - Command: /impeccable polish

4. **[P2] UI 언어 혼재 + 택소노미 명명 불일치**
   - What: "On This Page"/"MORE"/"Category"/404 영어 vs 한국어 본문; 사이드바 "Naver Bootcamp" vs 인덱스 "Naver Boost Camp AI Tech NLP Track"; 대소문자 혼재(AI/development/Obsidian/paper review); "토글 메뉴 +" 의미 불명.
   - Why: locale ko 선언에도 시스템 언어가 오락가락 — 신뢰·일관성 인상을 깎고 스크린리더 음성 전환을 유발.
   - Fix: UI 문자열 한국어 통일(_data/ui-text.yml), 카테고리 표기 규칙 1개로 통일, "토글 메뉴"→"카테고리 보기".
   - Command: /impeccable clarify

5. **[P2] 모바일 포스트 진입 프리앰블 ~2화면**
   - What: 브레드크럼(전체 제목 중복 2줄)+프로필+「토글 메뉴 +」+비접이식 전체 TOC를 지나야 첫 문단.
   - Why: 모바일 검색 유입 독자의 첫 5초를 잡아먹는다.
   - Fix: 모바일 브레드크럼 축약(또는 숨김), TOC 모바일 접기(details), 프로필을 글 하단으로.
   - Command: /impeccable adapt
